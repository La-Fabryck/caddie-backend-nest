# Oxlint / Oxfmt vs former ESLint + Prettier stack

This repo migrated static analysis to **Oxlint** ([`oxlint.config.mjs`](../oxlint.config.mjs)) and formatting to **Oxfmt** ([`.oxfmtrc.json`](../.oxfmtrc.json)). The former setup used `typescript-eslint` **strictTypeChecked**, `eslint-plugin-unicorn`, `eslint-plugin-perfectionist`, `eslint-plugin-unused-imports`, and **Prettier** via `eslint-plugin-prettier`.

## What was ported rule-for-rule

The explicit `rules` block from the old `eslint.config.mjs` is mirrored in [`oxlint.config.mjs`](../oxlint.config.mjs) using Oxlint’s `typescript/` rule IDs where applicable (see [consistent-type-imports](https://oxc.rs/docs/guide/usage/linter/rules/typescript/consistent-type-imports) and related TypeScript rules in the Oxc docs).

**Not the same as the old file:** the old config also spread `...tseslint.configs.strictTypeChecked`, which turns on **many** additional `@typescript-eslint` rules that are **not** listed one-by-one in [`oxlint.config.mjs`](../oxlint.config.mjs). Oxlint does not ship a drop-in “strictTypeChecked” bundle. To tighten further, add individual `typescript/*` rules from the [Oxlint rules list](https://oxc.rs/docs/guide/usage/linter/rules) or enable **`options.typeAware`** / install `oxlint-tsgolint` for type-aware rules.

## Other gaps

| Topic                             | Notes                                                                                                                                                          |
| --------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `typescript-eslint` strict preset | Not ported as a whole; only the explicit rule table above plus `categories.correctness` / `suspicious` as a coarse baseline.                                   |
| `eslint-plugin-perfectionist`     | Import sorting is handled by **Oxfmt** `sortImports` in [`.oxfmtrc.json`](../.oxfmtrc.json).                                                                   |
| `require-atomic-updates`          | **No Oxlint rule.**                                                                                                                                            |
| `test/tsconfig.json` override     | Former ESLint used a separate `parserOptions.project` for `test/**`. Oxlint resolves `tsconfig.json` per file when needed; no separate override is configured. |

## Oxfmt vs Prettier

- **Embedded formatting** in tagged templates may still differ from Prettier; see Oxfmt “unsupported features” in upstream docs.
- **`sortPackageJson`** is disabled in [`.oxfmtrc.json`](../.oxfmtrc.json) to avoid churn on `package.json`.

When tightening lint again, prefer **incremental** `typescript/*` additions over enabling whole categories at once.
